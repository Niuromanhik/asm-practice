# INT 0x80 в Linux (i386)

## Що таке int 0x80

int 0x80 — це програмне переривання в архітектурі x86 (i386), яке використовується в Linux для виконання системних викликів (system calls).

За допомогою int 0x80 користувацька програма може звернутися до ядра операційної системи для виконання операцій, які вона не може виконати самостійно (наприклад, робота з файлами або вивід у консоль).

## Як це працює

Перед викликом int 0x80:

- у регістр eax записується номер системного виклику
- у регістри ebx, ecx, edx, esi, edi записуються параметри

Після виконання переривання результат повертається в регістр eax.

## Найбільш часто використовувані системні виклики

### 1. sys_exit (номер 1)

Завершує програму.

- eax = 1
- ebx = код завершення

### 2. sys_write (номер 4)

Записує дані у файл або в консоль.

- eax = 4
- ebx = дескриптор файлу (1 = stdout)
- ecx = адреса буфера
- edx = кількість байтів

### 3. sys_read (номер 3)

Читає дані.

- eax = 3
- ebx = дескриптор (0 = stdin)
- ecx = адреса буфера
- edx = кількість байтів

### 4. sys_open (номер 5)

Відкриває файл.

- eax = 5
- ebx = ім'я файлу
- ecx = прапори відкриття
- edx = права доступу

### 5. sys_close (номер 6)

Закриває файл.

- eax = 6
- ebx = дескриптор файлу

## Висновок

int 0x80 — це механізм взаємодії програми з ядром Linux у 32-бітній архітектурі.

У сучасних 64-бітних системах замість int 0x80 використовується інструкція syscall.
